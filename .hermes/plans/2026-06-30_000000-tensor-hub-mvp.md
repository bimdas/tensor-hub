# Tensor Hub: On-Device AI Inference Server

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** An Android app that runs ML models locally on Pixel's Tensor TPU/GPU via LiteRT, exposing an OpenAI-compatible HTTP API for local consumption by Hermes and other tools.

**Architecture:** Android foreground service running a NanoHTTPD server. LiteRT runtime with NNAPI delegate for hardware acceleration. Model Manager handles downloading, caching, and lifecycle. Inference Engine wraps LiteRT interpreter with a unified API. HTTP server exposes endpoints compatible with OpenAI's API format so existing tools (whisper clients, embedding clients) work out of the box.

**Tech Stack:** Kotlin, Jetpack Compose (UI), LiteRT (TFLite successor), NanoHTTPD (HTTP server), Gradle (build), Material 3 (design)

## Phases

### Phase 1: Scaffold & Core Infrastructure
- Android project with Gradle, Compose, LiteRT dependencies
- Application class, manifest with foreground service + network permissions
- Basic Compose UI with Material 3 dark theme
- NanoHTTPD server skeleton with health endpoint

### Phase 2: Inference Engine
- LiteRT interpreter wrapper with delegate management
- Model Manager (download, cache, load/unload lifecycle)
- Audio processing pipeline (decode, resample, mel spectrogram)
- Whisper inference pipeline (encoder + decoder)

### Phase 3: API Server
- OpenAI-compatible /v1/audio/transcriptions endpoint
- OpenAI-compatible /v1/embeddings endpoint
- /v1/models management endpoint
- Multipart form data handling for audio upload

### Phase 4: Polish & Ship
- Foreground service with persistent notification
- Settings screen (port, delegate preference, model paths)
- Error handling, logging, health monitoring
- README, API docs, model compatibility guide

## Key Design Decisions

1. **OpenAI-compatible API** - Existing whisper clients and embedding tools work without modification
2. **NNAPI delegate first** - Falls back to GPU delegate, then CPU. Auto-benchmarks on first run.
3. **Foreground service** - Server stays alive when app is backgrounded
4. **Model-agnostic architecture** - Easy to add new models beyond Whisper
5. **Localhost only** - No external network binding by default (security)
6. **INT8 quantization** - Optimal for Tensor TPU. Models must be quantized.

## API Endpoints

### Health
```
GET /health
Response: { "status": "ok", "models_loaded": [...], "delegate": "nnapi" }
```

### Audio Transcription (OpenAI-compatible)
```
POST /v1/audio/transcriptions
Content-Type: multipart/form-data
Body: file=<audio>, model=whisper-base, language=en, response_format=json

Response: { "text": "transcribed text..." }
```

### Embeddings (OpenAI-compatible)
```
POST /v1/embeddings
Content-Type: application/json
Body: { "model": "all-MiniLM-L6-v2", "input": "text to embed" }

Response: { "data": [{ "embedding": [...] }], "model": "all-MiniLM-L6-v2", "usage": { "prompt_tokens": 5, "total_tokens": 5 } }
```

### Models
```
GET /v1/models
Response: { "data": [{ "id": "whisper-base", "owned_by": "tensor-hub" }] }

POST /v1/models/load
Body: { "model": "whisper-base" }

POST /v1/models/unload
Body: { "model": "whisper-base" }
```
